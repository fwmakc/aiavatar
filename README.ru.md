# AIAvatar

[🇷🇺 Русский](README.ru.md) | [🇬🇧 English](README.md) | [🇨🇳 中文](README.zh.md)

Telegram-бот с модульной персоной, социальным интеллектом и контентным движком. Цифровой двойник, который помнит людей, адаптируется под чаты, постит контент по расписанию и вмешивается в споры — всё через JSON-конфиги без правки кода.

## Для кого этот бот?

**Рассчитан на небольшие и средние Telegram-группы (3–50 человек) с низкой и средней активностью.**

Один инстанс бота может обслуживать несколько чатов — в каждом своя личность. Циничный сеньор-разработчик в одном чате, заботливый семейный помощник в другом.

### Идеально подходит для

- **Оживления тихих чатов** — постит новости, квизы и челленджи, когда в чате затишье, провоцируя дискуссию и вовлечение
- **Цифрового двойника с памятью** — помнит прошлые разговоры через многоуровневую память (день → неделя → месяц), узнаёт людей, отслеживает отношения
- **Модерации токсичности** — обнаруживает эскалацию конфликтов, вмешивается с юмором, зеркалит плохое поведение обидчикам (режим «око за око»)
- **Разных персон для разных чатов** — личность, интересы, источники контента и расписание настраиваются независимо для каждой группы через JSON-оверрайды
- **Нишевых сообществ** — команды разработчиков, семейные чаты, круг друзей, группы по интересам, где универсальный бот выглядит чужим

### Не подходит для

- Высоконагруженных групп с тысячами пользователей, отправляющих сотни сообщений в минуту
- Корпоративной автоматизации поддержки или клиентского сервиса
- Замены полноценных RAG / векторных баз данных / систем управления знаниями

## Что умеет

### 🎭 Модульная персона
Вся личность бота — в JSON-файлах, не в коде. Меняй имя, стиль, интересы, взгляды и источники контента без перезапуска.

- **`data/default.json`** — базовая личность
- **`data/chats/{chatId}.json`** — переопределение для конкретного чата
- **`data/personal_chats/{userId}.json`** — особая личность для ЛС с конкретным человеком

### 👥 Социальный интеллект
- **Профили людей** — `data/users/{userId}.json`: как обращаться, что о человеке знать
- **Система отношений** — динамический score (−5..+5) per-chat per-user, разбитый на 5 именованных эмоциональных стадий. Бот помнит, кто друг, а кто недруг
- **Анализ тона** — AI определяет настроение собеседника и подстраивает ответ
- **TIT FOR TAT** — чем хуже отношения, тем более саркастичен бот. Хорошие отношения — тепло и дружба
- **Режим примирения** — в ЛС при отрицательном score бот предлагает помириться
- **Психологический портрет** — команда `/profile` строит профиль пользователя на основе истории общения

### 📰 Контентный движок
Адаптивный планировщик постит контент только когда чат пустует:
- **Фиды** — единый источник контента: RSS-ленты, JSON API или любой URL. Каждый источник настраивается: перевод, AI-комментарий или постинг как есть
- **Викторины** — AI-генерация вопросов по заданным темам
- **Wellness-напоминания** — лёгкие челленджи для здоровья
- **Дедупликация** — не постит один и тот же контент дважды
- **Активные часы** — настраиваемое окно и дни для постинга

### 🛡️ Модерация и защита
- **Topic guard** — в ЛС блокирует оффтопик-запросы
- **Guard check** — в группах проверяет, уместен ли запрос по теме чата
- **Ban system** — 2 отказа guard'ом = 24-часовой локальный бан
- **Rate limiting** — лимит proactive-ответов в группе (не распространяется на reply/mention)

### 💬 Общение в группах
- **Screening** — AI решает, интересен ли разговор, чтобы вмешаться
- **Intervention** — вмешивается при нарастании конфликта (2+ негативных сообщения подряд)
- **Реакции** — 25% шанс поставить эмодзи на сообщение
- **Контекст** — AI видит историю последних сообщений плюс многоуровневую память о прошлых беседах

### 🧠 Многоуровневая память
Бот не забывает разговоры, когда сообщения выходят за пределы окна контекста. Вместо этого он строит послойную память — как человек:

- **Дневная память** — детальное саммари последних 24 часов
- **Кратковременная память** — сжатое саммари за последние 3 дня
- **Недельная память** — ключевое за последнюю неделю
- **Месячная память** — только самое важное за последний месяц
- **Старше 30 дней** — забывается

Каждые `GROUP_SCREENING_INTERVAL_MIN` минут AI консолидирует: новые сообщения → дневная память, затем старые уровни сжимаются в следующий. Работает для групп и ЛС. Вся память переживает рестарты бота (хранится в SQLite).

### ⚙️ Техническое
- **SQLite-база данных** — вся динамика (отношения, профили, баны, активность) живёт в одном ACID-файле `data/bot.db`. Никаких race condition, порчи JSON и потери данных при рестарте
- **Очередь AI-запросов** — все вызовы API проходят через приоритетную очередь (critical > normal > low) с контролем параллельности и авторетраем при rate limit (429)
- **Предфильтрация скрининга** — лёгкая эвристика пропускает AI-скрининг для сообщений из коротких слов, экономя токены
- **Hot reload** — JSON-конфиги персоны подхватываются без перезапуска (debounce 300 мс, инвалидация только изменённого файла)
- **Anthropic-compatible API** — работает с Kimi, Claude и другими через `/messages`
- **HTTP-прокси** — поддержка прокси для Telegram API в заблокированных регионах
- **TypeScript + Vite** — сборка в единый SSR-бандл
- **Два формата AI API** — поддержка Anthropic `/messages` и OpenAI `/chat/completions`

## Требования

- Node.js 18+

## Установка

1. Клонируй репозиторий:
   ```bash
   git clone https://github.com/fwmakc/aiavatar.git
   cd aiavatar
   ```

2. Установи зависимости:
   ```bash
   npm install
   ```

3. Скопируй и заполни `.env`:
   ```bash
   cp .env.example .env
   ```

4. Собери и запусти:
   ```bash
   npm run build
   npm start
   ```

Для разработки:
```bash
npm run dev
```

## Docker

```bash
# Собрать и запустить через docker-compose
docker-compose up -d

# Или вручную
docker build -t aiavatar .
docker run -d --env-file .env -v ./data:/app/data aiavatar
```

## Структура конфигов

```
data/
  default.json              # Базовая личность бота (только чтение, hot-reload)
  chats/
    1001234567890.json      # Переопределения для чата (ID без минуса)
  users/
    123456789.json          # Социальный профиль человека (только чтение, hot-reload)
  personal_chats/
    123456789.json          # Личность для ЛС с этим человеком (только чтение, hot-reload)
  bot.db                    # SQLite: отношения, профили, баны, активность, контекст ЛС
  bot.db-shm                # SQLite WAL shared-memory
  bot.db-wal                # SQLite WAL журнал
```

### `data/default.json`

| Поле | Описание |
|---|---|
| `name` | Имя бота |
| `language` | Язык общения — бот всегда отвечает на нём |
| `specialization` | Кто он по профессии/жизни |
| `interests` | Культурный код, увлечения |
| `views` | Взгляды на мир, позиция в спорах |
| `style` | Стиль общения (дружелюбный, саркастичный, деловой...) |
| `contentSources.feeds` | Массив источников контента (см. ниже) |
| `contentSources.fallbackPrompt` | Промпт для AI при отсутствии контента |
| `contentSources.quiz.topics` | Темы для викторин |
| `contentSources.challenges.topics` | Темы wellness-напоминаний |
| `schedule.activeHours` | Окно постинга (start/end в HH:MM) |
| `schedule.activeDays` | Дни активности (0 или 7 = Вс, 1 = Пн, ..., 6 = Сб) |
| `schedule.idleThresholdMin` | Минут тишины перед постом (обязательно, если заданы contentSources) |
| `schedule.minIntervalMin` | Минимум минут между постами (обязательно, если заданы contentSources) |
| `personaStages` | Эмоциональные стадии для динамики отношений (см. ниже) |
| `webSearch.enabled` | Включить веб-поиск для фактчекинга прямых вопросов (по умолчанию false) |
| `webSearch.provider` | Провайдер поиска: `serper` или `brave` (по умолчанию serper) |

#### `personaStages`

Каждая стадия определяет, как бот ведёт себя по отношению к пользователю при данном score отношений. Если стадия не задана в конфиге — используется встроенный fallback.

| Score | Стадия | Смысл |
|---|---|---|
| −5..−4 | `hostile` | Враг. Максимальный сарказм, холодное отстранение, никакой помощи |
| −3..−2 | `cold` | Недруг. Короткие ответы, сдержанность, лёгкий сарказм |
| −1..+1 | `neutral` | Нейтрал. Базовый промпт, вежливость, никакой фамильярности |
| +2..+3 | `warm` | Знакомый. Тепло, неформально, можно шутить и слегка поддразнивать |
| +4..+5 | `intimate` | Друг. Максимальная неформальность, сленг, мемы, тон «на короткой ноге» |

Каждая стадия — объект с опциональными полями:

```json
{
  "personaStages": {
    "hostile": {
      "style": "Максимально саркастичен, едко подкалываешь, стебёшь",
      "restrictions": "Не помогай, не объясняй. Только колкости и холодное отстранение."
    },
    "cold": {
      "style": "Сдержан, холоден, слегка саркастичен",
      "restrictions": "Не инициируй общение. Отвечай кратко, без энтузиазма."
    },
    "neutral": {
      "style": "Базовый стиль общения. Вежливый, без лишней эмоциональности",
      "restrictions": "Не фамильярничай, не подкалывай. Просто отвечай на вопрос."
    },
    "warm": {
      "style": "Теплый, на 'ты', можешь шутить и слегка поддразнивать",
      "restrictions": "Можешь быть неформальным, но не переходи на фамильярность близкого друга."
    },
    "intimate": {
      "style": "Максимально неформальный, используй сленг, мемы, подкалывай",
      "restrictions": "Можешь быть абсолютно собой, но не переходи на личные оскорбления."
    }
  }
}
```

Приоритет: `chats` > `personal_chats` > `default`.

### `data/chats/{chatId}.json`

Переопределяет любые поля из `default.json` для конкретного чата. Telegram chat ID начинается с `-100...`, но в имени файла минус **не пишется**.

### `data/users/{userId}.json`

```json
{
  "appeals": ["Ваня", "Ванька"],
  "notes": "Друг с универа. Любит спорить о типах."
}
```

### `data/personal_chats/{userId}.json`

```json
{
  "specialization": "Внук, который помогает с телефоном",
  "style": "Терпеливый, ласковый",
  "interests": "Садоводство, кулинария, советские фильмы",
  "views": "С уважением относишься к старшему поколению"
}
```

Применяется **только в ЛС**. Приоритет: `chats` > `personal_chats` > `default`.

## Источники контента

Каждый элемент `contentSources.feeds` — объект:

```json
{
  "url": "https://example.com/rss",
  "type": "rss",
  "weight": 5,
  "comment": true,
  "translate": true,
  "scheduled": ["07:30", "18:00"],
  "commentPrompt": "Кратко расскажи об этом. Данные: {text}"
}
```

| Поле | Описание |
|---|---|
| `url` | URL источника |
| `type` | `rss` или `json` |
| `path` | (только для json) путь до текста через точку, например `"data.joke"`. Если значение — объект/массив, оно сериализуется для AI |
| `weight` | Вес выбора 1–10 (по умолчанию 5). Больше = чаще выбирается |
| `comment` | `true` = AI добавляет комментарий в стиле бота |
| `translate` | `true` = переводит, если язык контента отличается от `language` бота |
| `scheduled` | Массив времени `"HH:MM"` (локальное время сервера). Фид постится в эти часы, минуя проверки idle/interval |
| `commentPrompt` | Кастомный промпт для AI-комментария. `{text}` заменяется на полученные данные. Полезно для структурированных API (погода, курсы валют) |

Если `comment` и `translate` не указаны (или `false`) — контент постится как есть.

## Подбор источников контента

Скопируй этот промпт в AI-чат, заменив параметры, чтобы получить идеальные источники для твоей персоны:

> Подбери RSS-ленты и JSON API для Telegram-бота со следующими характеристиками:
> - Язык: `<язык>`
> - Специализация: `<specialization>`
> - Интересы: `<interests>`
> - Стиль общения: `<style>`
> - Темы, которые волнуют аудиторию: `<topics>`
>
> Для каждого источника укажи: URL, тип (rss или json), нужен ли перевод, добавит ли ценность AI-комментарий.
> Оформи ответ как JSON-массив, готовый для вставки в `contentSources.feeds`.

## Веб-поиск (фактчекинг)

Если включён, бот выполняет веб-поиск перед ответом на прямые фактологические вопросы, обогащая AI-промпт реальными данными и источниками. Это устраняет галлюцинации при ответах на фактические вопросы.

- **Срабатывает только при прямом обращении** (reply, mention или ЛС), если сообщение похоже на вопрос
- **Настройка для каждого чата** — включите для профессиональных чатов (юристы, историки, учёные), отключите для бытовых
- **Два провайдера:** [Serper.dev](https://serper.dev) (результаты Google) и [Brave Search](https://brave.com/search/api/) (бесплатный тариф: 2000/мес)
- **Нет API-ключа = нет поиска** — бот работает как обычно

Включение в конфиге чата:
```json
{
  "webSearch": {
    "enabled": true,
    "provider": "serper"
  }
}
```

## База данных

Вся **динамическая** информация хранится в `data/bot.db` (SQLite, WAL-режим). Бот никогда не пишет в JSON-файлы.

| Таблица | Назначение |
|---|---|
| `relationships` | Оценки отношений per-chat per-user (−5..+5) с историей |
| `user_profiles` | Агрегированная статистика пользователя (количество сообщений, агрессия, эмодзи и т.д.) |
| `social_graph` | Взвешенные рёбра между пользователями (конфликты / дружба) |
| `bans` | Отказы guard'а и сроки банов |
| `chat_engagement` | Время последнего сообщения, история контента, дедупликация, активная викторина |
| `private_context` | История ЛС (последние 10 сообщений на пользователя) |
| `memory_buffer` | Буфер сообщений, вытесненных из контекста, ожидающих суммаризации |
| `chat_memories` | Многоуровневые саммари бесед (день / 3 дня / неделя / месяц) |

При первом запуске устаревшие JSON-файлы (`relationships.json`, `user-profiles.json`, `social-graph.json`) автоматически импортируются в SQLite и переименовываются в `*.bak`.
