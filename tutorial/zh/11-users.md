# 11. 人物档案

人物档案让机器人记住谁是谁——如何称呼、了解什么信息、关系如何。

## 如何获取 user ID

### 方法 1：通过机器人

在聊天中添加 [@userinfobot](https://t.me/userinfobot)。将对方的消息转发给它——它会显示 user ID。

### 方法 2：通过控制台

当有人给机器人发消息时，控制台会显示：
```
[DM] 123456789: 你好！
```
`[DM]` 后面的数字就是 user ID。

### 方法 3：通过 Telegram API

向机器人发送任意消息，然后在浏览器中打开：
```
https://api.telegram.org/bot你的令牌/getUpdates
```
找到 `"from":{"id":123456789}`——这就是 user ID。

## 如何获取群组的 chat ID

在已添加机器人的群组中发送一条消息，然后：
```
https://api.telegram.org/bot你的令牌/getUpdates
```
找到 `"chat":{"id":-1001234567890}`——这就是 chat ID（带负号）。

在配置文件名中**去掉负号**：`1001234567890`。

## 创建档案

创建文件 `data/users/{userId}.json`：

```json
{
  "appeals": ["小明", "明哥"],
  "notes": "前端开发者。喜欢 React 和 TypeScript。咖啡喝得很多。周五喜欢玩桌游。"
}
```

### 字段说明

| 字段 | 描述 | 示例 |
|---|---|---|
| `appeals` | 如何称呼对方 | `["小明", "明哥", "小明同学"]` |
| `notes` | 关于对方的信息 | `"前端开发，喜欢 React"` |

机器人会使用 `appeals` 来称呼对方，使用 `notes` 在对话中提供上下文。

## 档案示例

### 同事（"小王"场景）

```json
{
  "appeals": ["小王", "王哥"],
  "notes": "Python 后端开发者。喜欢争论类型系统。Linux 粉丝。每天早上从读 Hacker News 开始。"
}
```

### 家庭成员（"小雨"场景）

```json
{
  "appeals": ["小雨", "雨宝"],
  "notes": "女儿，8 岁。上二年级。喜欢画画和猫。周三上舞蹈课。怕黑。"
}
```

## 私聊中的个性化人格

详见[第 8 章](08-personal-chats.md)——其中有针对孩子、奶奶和同事的示例。

简而言之：创建 `data/personal_chats/{userId}.json`，机器人就会与特定人物在私聊中使用独特的方式交流。这**不影响**群组中的行为。

## 关系系统

机器人会自动与每个人建立关系：

- **分数范围 −5 到 +5**——取决于对方与机器人的互动方式
- **以牙还牙：** 对机器人粗鲁——机器人冷淡回应；提供帮助——机器人热情回应
- **5 个阶段：** hostile → cold → neutral → warm → intimate

关系数据存储在数据库（`data/bot.db`）中，无需手动配置。
