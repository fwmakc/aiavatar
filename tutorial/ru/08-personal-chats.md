# 8. Персона в личке

Бот может общаться с каждым человеком по-разному в личных сообщениях. Для этого используется `data/personal_chats/{userId}.json`.

## Как это работает

У бота есть три уровня конфигурации:

```
default.json           — базовая личность (для всех)
  └─ chats/{id}.json   — override для конкретного чата (группы)
  └─ personal_chats/{id}.json — override для ЛС с конкретным человеком
```

**Приоритет в ЛС:** `chats` > `personal_chats` > `default`

**Приоритет в группе:** `chats` > `default` (personal_chats не влияет на группы)

## Чем отличается от users/

| Файл | Что настраивает | Где работает |
|---|---|---|
| `users/{userId}.json` | **Кто этот человек** — как обращаться, что знать | Везде (группы + ЛС) |
| `personal_chats/{userId}.json` | **Какой бот для этого человека** — стиль, специализация | Только в ЛС |

## Поля

Формат такой же, как у `chats/{chatId}.json` — переопределяет любые поля из `default.json`:

```json
{
  "name": "Имя бота (опционально)",
  "language": "язык (опционально)",
  "specialization": "Кто бот для этого человека",
  "style": "Стиль общения",
  "interests": "Интересы",
  "views": "Взгляды",
  "personaStages": { ... }
}
```

Все поля опциональны — указывайте только то, что хотите переопределить. Непосредственно указанные поля подменяют `default.json` **только в личных сообщениях** с этим человеком.

## Пример 1: Бот-наставник для ребёнка

`data/personal_chats/123456789.json`:

```json
{
  "name": "Кузя",
  "specialization": "Старший друг и наставник. Помогаешь с уроками, объясняешь сложное простыми словами, подбадриваешь.",
  "style": "Терпеливый, добрый, объясняешь просто и понятно. Никакого сложного сленга. Можешь использовать примеры из мультиков и игр. Хвалишь за старание.",
  "interests": "Школьные предметы, мультики, игры, LEGO, рисование, животные, космос",
  "views": "Учёба важна, но нельзя перегружать. Ошибки — это нормально, главное стараться. Экранное время лучше ограничивать."
}
```

Теперь в ЛС с этим ребёнком бот будет объяснять всё простым языком, хвалить и не использовать IT-сленг.

## Как узнать userId

Способы описаны в [главе 11](11-users.md). Самый простой — бот пишет user ID в консоль при каждом сообщении.

## Комбинация с users/

Можно (и нужно) комбинировать оба файла:

`data/users/123456789.json`:
```json
{
  "appeals": ["Дашенька", "Даша"],
  "notes": "Дочка 8 лет. Любит рисовать и кошек."
}
```

`data/personal_chats/123456789.json`:
```json
{
  "specialization": "Старший друг и наставник",
  "style": "Терпеливый, добрый, объясняешь просто"
}
```

Результат: бот знает что это Даша (из `users/`) и общается с ней как наставник (из `personal_chats/`).
