# 11. Полезные советы

## Hot reload — меняем конфиг без перезапуска

Бот автоматически подхватывает изменения в JSON-файлах:

- `data/default.json` — базовая личность
- `data/chats/{chatId}.json` — конфиг чата
- `data/personal_chats/{userId}.json` — персональная личность
- `data/users/{userId}.json` — профили людей

**Просто отредактируйте файл и сохраните** — бот применит изменения через 300 мс.

`.env` **не поддерживает** hot reload — для изменения переменных окружения нужен перезапуск.

## Proxy — если Telegram заблокирован

Добавьте в `.env`:

```env
PROXY_URL=http://user:password@host:port
```

Поддерживаются HTTP и SOCKS5 прокси. Формат:
- HTTP: `http://user:pass@proxy.example.com:8080`
- SOCKS5: `socks5://user:pass@proxy.example.com:1080`

## Обновление бота

```bash
git pull
npm install
npm run build
```

Затем перезапустите бота. База данных (`data/bot.db`) и JSON-конфиги не затрагиваются.

## Логирование

Бот пишет в консоль все свои действия:

```
[Persona] default.json loaded
[ContentEngine] Started (adaptive)
[DM] 123456789: Привет!
[AI] Response sent to 123456789
[Group] chat -1001234567890: screening...
[ContentEngine] chat 1001234567890: posted feed
```

Для сохранения логов в файл:
```bash
npm start 2>&1 | tee bot.log
```

Или через systemd (если запускаете как службу) — журнал автоматически сохраняется в `journalctl`.

## Контроль доступа

### Ограничить по пользователям

В `.env` укажите разрешённые user ID через запятую:

```env
ALLOWED_USERS=123456789,987654321
```

Пустое значение = бот отвечает всем.

Бот автоматически регистрирует чаты, в которые его добавили. Контентный движок работает только в зарегистрированных чатах.

## Как отключить отдельные функции

Всё управляется через `.env`:

| Переменная | Что отключает |
|---|---|
| `CONTENT_ENGINE_ENABLED=false` | Не постит контент (новости, шутки, викторины) |
| `GROUP_ACTIVE_MODE=false` | Не вмешивается в групповые разговоры |
| `GUARD_ENABLED=false` | Не проверяет темы запросов |
| `TIT_FOR_TAT_MODE=false` | Отношения не влияют на стиль общения |

## Веб-поиск для фактчекинга

Бот может выполнять поиск в интернете перед ответом на фактологические вопросы. Это исключает галлюцинации — AI видит реальные данные и источники.

### Когда срабатывает поиск

Поиск **НЕ** происходит на каждое сообщение. Только если **одновременно** выполнены все условия:

1. **Прямое обращение** — бот адресован лично (reply, mention или ЛС). На обычные сообщения в группе — не срабатывает
2. **Сообщение похоже на вопрос** — содержит `?` или вопросительные слова (что, как, почему, какой, where, what, how...)
3. **Включён в конфиге** — `webSearch.enabled: true` для данного чата
4. **Есть API-ключ** — `SERPER_API_KEY` или `BRAVE_API_KEY` в `.env`

Приветствия, шутки, обсуждения,Small talk — всё это **не** триггерит поиск. Только прямые вопросы.

### Сценарий: юрист-консультант

Представьте чат юристов, где бот выступает как коллега-консультант. Без фактчекинга AI может выдумать номер статьи или несуществующую ссылку. С поиском — он сначала находит реальную статью, а потом формулирует ответ.

**1. Получите API-ключ**

Зарегистрируйтесь на [Serper.dev](https://serper.dev) (Google-результаты) или [Brave Search](https://brave.com/search/api/) (бесплатный тариф 2000/мес).

Добавьте ключ в `.env`:

```env
SERPER_API_KEY=ваш_ключ_здесь
```

**2. Создайте конфиг чата**

Файл `data/chats/{chatId}.json` для чата юристов:

**3. Как это работает на практике**

Пользователь спрашивает: *"Какая ответственность за задержку зарплаты?"*

1. Бот определяет: прямое обращение + вопрос → запуск поиска
2. Поиск: `"ответственность за задержку зарплаты"` → топ-3 результата с КонсультантПлюс, Гарант, pravo.ru
3. Результаты добавляются в промпт: *"Web search results: Статья 236 ТК РФ..."*
4. AI формулирует ответ, опираясь на реальные данные, и вплетает ссылку естественно

Пользователь спрашивает: *"Как дела?"* — поиск **не** срабатывает, бот отвечает как обычно.

**4. Два провайдера**

| Провайдер | Результаты | Цена | Ключ в .env |
|---|---|---|---|
| `serper` | Google | $50/50k запросов | `SERPER_API_KEY` |
| `brave` | Brave Search | 2000/мес бесплатно | `BRAVE_API_KEY` |

Укажите провайдера в конфиге: `"provider": "serper"` или `"provider": "brave"`.
